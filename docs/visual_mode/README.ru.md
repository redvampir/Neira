# Рабочий процесс Visual Mode

Visual Mode Neira извлекает структурированные графы из аннотированного исходного кода. Рабочий процесс:

1. **Аннотируйте** код с помощью комментариев `@neyra:`, описывающих блоки, переменные и их связи.
2. **Парсите** исходник с помощью парсера визуального режима для построения метаданных.
3. **Редактируйте** полученный граф в визуальном редакторе.
4. **Локализуйте** отображаемые имена через поля `i18n.<locale>`, когда нужны переводы.

## Синтаксис аннотаций
Аннотации — это обычные комментарии, начинающиеся с `@neyra:` и содержащие тип и разделённые пробелами пары `ключ=значение`.

Типичные аннотации:

- `@neyra:visual_block` — помечает блок, видимый в редакторе.
- `@neyra:var` — объявляет переменную или параметр.
- `@neyra:connection` — связывает два идентификатора.

Строки с пробелами можно заключать в кавычки `"`. Дополнительные атрибуты допустимы и игнорируются парсерами, которые их не понимают.

## Поддерживаемые языки
Парсер поставляется с модулями для:
C, C++, C#, Dart, Go, Haskell, Java, JavaScript, Kotlin, MATLAB, Objective-C, PHP, Python, R, Ruby, Rust, Scala, Swift и TypeScript. Любой язык, поддерживающий комментарии, может содержать аннотации.

## Рекомендации для переводчиков

- Добавляйте локализованные имена через `i18n.<locale>="Перевод"`.
- Не изменяйте значения `id` и сохраняйте семантику кода.
- Указывайте авторство с помощью атрибутов `translator="Имя"` и `reviewer="Имя"`.

## Использование AI‑метаданных

Если аннотации или переводы созданы автоматически, добавьте `source=ai` и при желании `model="gpt-4"` или `confidence=0.9`. Для материалов, подготовленных вручную, используйте `source=human` и укажите `translator`.

Языко‑специфические примеры находятся в папке [examples](examples/).
