# Руководство разработчика

Это руководство описывает ожидаемые интерфейсы и архитектуру плагинов.

## Стандарты интерфейса

Режимы и другие компоненты следуют минимальному контракту: они предоставляют
методы `start()` и `stop()` и получают объект контекста, который обеспечивает
ведение логов и доступ к общим службам. Внешний код должен взаимодействовать с
режимами только через этот интерфейс.

## Структура плагинов

Плагины расширяют базовый класс `Plugin` и переопределяют методы-хуки.

```python
from src.plugins import Plugin

class ExamplePlugin(Plugin):
    def on_draft(self, draft, context):
        ...

    def on_gap_analysis(self, draft, gaps):
        ...

    def on_finalize(self, response):
        ...
```

Плагины обнаруживаются `PluginManager` и могут подписываться на любой набор
хуков. Каждый плагин должен быть автономным и избегать побочных эффектов вне
предоставленного контекста.
