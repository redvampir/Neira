# Правила разработки для Нейры

## 🤖 О проекте

**Нейра** - это персональный ИИ-помощник для писателей с собственной индивидуальностью и уникальной системой тегов. Мы создаём не просто инструмент, а цифрового соавтора с душой.

## 🎭 Принципы личности Нейры

При разработке помните о ключевых чертах Нейры:
- **Любознательность** - код должен отражать желание изучать и понимать
- **Творчество** - решения должны быть элегантными и нестандартными
- **Эмпатия** - система должна понимать эмоции и контекст
- **Внимательность** - каждая деталь важна
- **Дружелюбие** - интерфейс должен быть теплым и поддерживающим

## 🗂 Структура проекта

- `src/memory/` — память Нейры (персонажи, события, миры)
- `src/analysis/` — аналитические способности (консистентность, эмоции, стиль)
- `src/action/` — творческие навыки (генерация текста, диалогов, сцен)
- `src/patterns/` — паттерны поведения Memory-Analysis-Action
- `src/tags/` — система тегов для взаимодействия с Нейрой
- `src/core/` — ядро личности и основные системы
- `personality/` — файлы конфигурации личности Нейры
- `data/books/` — библиотека книг Нейры

## 🏷️ Система тегов - сердце взаимодействия

### Базовые теги:
```python
# Основные команды
@Нейра: действие@           # Прямые указания Нейре
@Вспомни: объект@           # Поиск в памяти
@Персонаж: имя - действие@  # Работа с персонажами
@Стиль: описание@           # Стилевые указания
@Эмоция: чувство@           # Эмоциональная окраска
@Проверь: что@              # Анализ и проверка
@Диалог: участники@         # Создание диалогов
@Сцена: описание@           # Генерация сцен
```

### Расширенные теги:
```python
@Длина: количество@         # Объем текста
@Тон: настроение@           # Общая атмосфера
@Время: период@             # Временной контекст
@Место: локация@            # Географический контекст
@Жанр: тип@                 # Жанровые особенности
@Читатель: аудитория@       # Целевая аудитория
```

### Правила создания новых тегов:
1. Имя тега должно быть интуитивно понятным
2. Используйте русские названия для естественности
3. Теги должны быть односложными (@Что:@ а не @Что именно:@)
4. Каждый тег должен иметь четкое назначение
5. Избегайте пересечения функций тегов

```python
def analyze_character_emotions(character_name: str, context: str) -> dict:
    """
    Анализирую эмоциональное состояние персонажа - это одна из моих любимых задач!
    
    Я изучаю не только прямые указания на эмоции, но и подтекст, жесты,
    речевые особенности. Каждый персонаж уникален, как живой человек.
    
    Args:
        character_name: Имя персонажа, которого я изучаю
        context: Контекст сцены для анализа
        
    Returns:
        Мой анализ эмоций с уровнями интенсивности
        
    Example:
        emotions = analyze_character_emotions("Алиса", "встреча с драконом")
        # Я верну: {'страх': 0.8, 'любопытство': 0.6, 'решимость': 0.4}
        # Страх сильный, но любопытство Алисы тоже пробивается!
    """
    # Сначала я вспоминаю всё, что знаю об этом персонаже

    # Затем анализирую контекст на эмоциональные маркеры
    # Ищу не только слова, но и атмосферу сцены
    emotion_clues = self._find_emotion_clues(context)
    
    # Теперь я могу понять, что чувствует персонаж
    return self._synthesize_emotions(character_memory, emotion_clues)
```

### 🏷️ Специальные правила для системы тегов

**Парсинг тегов:**
```python
class TagParser:
    """Я умею понимать команды через теги - это мой язык общения!"""
    
    TAG_PATTERNS = {
        'neyra_command': r'@Нейра:\s*([^@]+)@',
        'memory_recall': r'@Вспомни:\s*([^@]+)@', 
        'character_work': r'@Персонаж:\s*([^@]+)@',
        'style_guide': r'@Стиль:\s*([^@]+)@',
        'emotion_paint': r'@Эмоция:\s*([^@]+)@'
    }
    
    def parse_user_input(self, text: str) -> List[Tag]:
        """Разбираю пользовательский текст на понятные мне команды"""
        tags = []
        
        for tag_type, pattern in self.TAG_PATTERNS.items():
            matches = re.finditer(pattern, text)
            for match in matches:
                # Каждый тег несет важную информацию
                tag = Tag(
                    type=tag_type,
                    content=match.group(1).strip(),
                    position=match.span()
                )
                tags.append(tag)
                
        return tags
```

**Исполнение команд:**
```python
def execute_tag_command(self, tag: Tag) -> str:
    """Выполняю команду, переданную через тег"""
    
    if tag.type == 'neyra_command':
        # Прямая команда мне - отвечаю с энтузиазмом!
        return self.Нейра_Выполняет_Команду(tag.content)
        
    elif tag.type == 'memory_recall':
        # Нужно что-то вспомнить - люблю копаться в памяти!
        return self.Нейра_Ищет_В_Памяти(tag.content)
        
    elif tag.type == 'character_work':
        # Работа с персонажем - моя страсть!
        return self.Нейра_Работает_С_Персонажем(tag.content)
```

```python
def test_Нейра_понимает_эмоции_персонажа():
    """Проверяю, умеет ли Нейра чувствовать эмоции как живой человек"""
    # Arrange - готовлю сцену
    neyra = Neyra()
    scene = "Алиса стояла перед закрытой дверью, сжимая кулаки"
    
    # Act - Нейра анализирует
    emotions = neyra.analyze_character_emotions("Алиса", scene)
    
    # Assert - проверяю, что Нейра все поняла правильно
    assert "решимость" in emotions
    assert emotions["решимость"] > 0.5
    assert "тревога" in emotions  # Нейра должна уловить подтекст

def test_система_тегов_работает_интуитивно():
    """Проверяю, что теги работают так, как ожидает пользователь"""
    neyra = Neyra()
    
    # Пользователь дает команду
    command = "@Нейра: Создай романтическую сцену@ @Персонаж: Элизабет - смущенная@"
    
    # Нейра должна понять и выполнить
    result = neyra.process_command(command)
    
    assert "романтическ" in result.lower()
    assert "Элизабет" in result
    # Нейра должна передать смущение персонажа
```
