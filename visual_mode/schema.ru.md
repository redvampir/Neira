# Аннотации визуального режима в комментариях

Визуальный режим Neira считывает метаданные, встроенные в комментарии кода. Каждая аннотация начинается с префикса `@neyra:`, чтобы базовый парсер языка мог их игнорировать.

## Общие правила
- Аннотации должны находиться внутри синтаксиса комментариев языка-хоста (например, `#`, `//`, `/* */`).
- Размещайте каждую аннотацию на отдельной строке или в конце строки кода.
- Атрибуты записываются в виде пар `ключ=значение`, разделённых пробелами. Строки могут быть заключены в кавычки `"` при наличии пробелов.

Дополнительные аннотации можно определять с тем же префиксом `@neyra:`.

## `@neyra:visual_block`
Помечает участок кода, представляющий блок в визуальном редакторе.

Пример:
```python
# @neyra:visual_block id="sum" display="Add" category="math"
```

**Поля**
- `id` (обязательно) — уникальный идентификатор блока.
- `display` — основное имя, отображаемое в интерфейсе.
- `i18n.<locale>` — локализованные имена, например `i18n.es="Suma"`.
- `category` — строка для группировки блоков.
- `range` — `start_line:start_col-end_line:end_col`, описывающий охваченный код.

## `@neyra:var`
Объявляет переменную или параметр, используемые в визуальном графе.

```
# @neyra:var id="x" display="X value"
```

Поля аналогичны `@neyra:visual_block` и могут включать `category`, `display`, `i18n.<locale>` и `range`.

## `@neyra:connection`
Определяет соединение между блоками или переменными.

```
# @neyra:connection from="sum" to="result" category="data"
```

**Поля**
- `from` и `to` (обязательно) — идентификаторы узла-источника и узла-назначения.
- `display`, `i18n.<locale>` и `category` — необязательные метаданные.

## Кодирование позиции
Позиции и диапазоны используют нумерацию строк и столбцов с единицы.
- `position=line:column` обозначает одну точку.
- `range=start_line:start_col-end_line:end_col` обозначает диапазон.

## Имена и локализация
- `display` задаёт имя по умолчанию.
- `i18n.<locale>` переопределяет имя для указанной локали (например, `i18n.fr`).

## Категории
Поле `category` группирует блоки, переменные или соединения по произвольной строке.

## Интеграция с редакторами
Поскольку аннотации — обычные комментарии, редакторы, которые их не поддерживают, должны полностью их игнорировать. Инструменты, понимающие визуальный режим, могут разбирать строки `@neyra:`, но не должны изменять семантику программы при добавлении или удалении аннотаций.
